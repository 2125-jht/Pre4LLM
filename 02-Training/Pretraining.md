# 2. 预训练技术 (Pretraining)

## 2.1 预训练八股

### 2.1.1 预训练的核心目标是什么？与微调的本质区别？

**参考答案：**

**预训练核心目标：**
- 从海量无标注文本学习**通用语言表示**
- 掌握语法、语义和世界知识
- 获得强大的**迁移学习能力**

**本质区别：**

| 维度 | 预训练 | 微调 |
|------|--------|------|
| 数据 | 无标注、TB级 | 有标注、GB级 |
| 目标 | 下一个token预测 | 特定任务适配 |
| 学习率 | 较大 (1e-4) | 较小 (1e-5) |
| 本质 | 学习如何学习 | 学习具体任务 |

---

### 2.1.2 预训练数据的来源有哪些？各有什么特点？

**参考答案：**

| 来源 | 特点 | 作用 |
|------|------|------|
| **网页数据** | 量大、噪声多、覆盖广 | 基础语言知识 |
| **书籍** | 质量高、长距离依赖 | 叙事能力 |
| **百科** | 事实准确、结构化 | 事实性知识 |
| **代码** | 逻辑严密、结构化 | 推理能力 |
| **对话** | 口语化、交互性 | 对话能力 |

**数据配比原则：** 高质量数据（如Wikipedia）可适当上采样，代码数据对推理能力提升显著。

---

### 2.1.3 数据清洗的关键步骤有哪些？

**参考答案：**

```
原始数据 → 格式标准化 → 语言过滤 → 质量评分 → 去重 → 敏感过滤 → 训练数据
```

**关键步骤：**
1. **格式标准化**：去HTML标签、统一编码
2. **质量评分**：基于困惑度、规则过滤
3. **去重**：MinHash/LSH去重，防止数据污染
4. **敏感过滤**：PII信息、有害内容

**为什么重要：** Garbage In, Garbage Out。低质量数据会导致模型学到错误知识。

---

### 2.1.4 BPE、WordPiece、SentencePiece的区别？

**参考答案：**

| 特性 | BPE | WordPiece | SentencePiece |
|------|-----|-----------|---------------|
| 合并策略 | 频率最高 | 似然最大 | 可选BPE/Unigram |
| 预分词 | 需要 | 需要 | **不需要** |
| 多语言 | 一般 | 一般 | **优秀** |
| 代表模型 | GPT系列 | BERT | T5、LLaMA |

**关键区别：**
- **BPE/WordPiece**：需要预分词（空格分词），对中文不友好
- **SentencePiece**：直接处理原始文本，语言无关，适合多语言模型

### 2.1.5 什么是Scaling Laws？对实际训练有什么指导意义？

**参考答案：**

**核心发现：** 模型性能与参数量、数据量呈**幂律关系**

```
Loss(N, D) = E + A/N^α + B/D^β
```

**Chinchilla最优条件：**
- Token数 ≈ **20 × 参数量**
- 例如：7B模型最优训练数据量 = **140B tokens**

**实际指导：**
1. 给定计算预算，模型大小和数据量应**等比例增长**
2. 小模型+大数据 > 大模型+小数据
3. 过度训练（Over-training）在某些任务上有益

---

### 2.1.6 混合精度训练的原理和优势？

**参考答案：**

**原理：** 训练时同时使用FP16和FP32
- **FP16**：前向/反向传播计算（速度快、省显存）
- **FP32**：参数更新（精度保持）

**FP16的问题与解决：**

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 梯度下溢 | FP16表示范围小 | **Loss Scaling** |
| 舍入误差 | 精度有限 | 关键操作保留FP32 |

**BF16 vs FP16：**
- **BF16**：指数位8位（与FP32相同），无需Loss Scaling，A100+推荐
- **FP16**：尾数位更多，精度更高，但需要Loss Scaling

---

### 2.1.7 梯度裁剪的作用？什么情况下需要使用？

**参考答案：**

**作用：** 限制梯度最大范数，防止梯度爆炸

```
if ||g|| > max_norm:
    g = g × (max_norm / ||g||)
```

**使用场景：**
1. 使用**RNN/LSTM**（必须）
2. **大学习率**训练
3. **混合精度**训练（额外安全保障）
4. **长序列**训练

**典型值：** max_norm = 1.0

---

### 2.1.8 什么是Gradient Checkpointing？

**参考答案：**

**原理：** 以**计算换显存**
- 只保存部分检查点的激活值
- 反向传播时重新计算中间激活

**显存节省：** 从O(N)降到O(√N)，可节省约**90%激活显存**

**代价：** 增加约**20-30%**计算时间

**适用场景：** 显存不足、长序列训练、大模型微调

---

### 2.1.9 预训练中的学习率调度策略？

**参考答案：**

**常用策略：** Linear Warmup + Cosine Decay

```
学习率
  ↑
  │      ╭────╮
  │     ╱      ╲____
  │    ╱            ╲___
  │___╱                  ╲
  └────────────────────────→ 步数
    Warmup    Decay
```

**Warmup作用：**
1. 训练初期参数随机，大学习率导致梯度爆炸
2. 让Adam的动量估计稳定
3. 典型比例：占总步数的0.5%-2%

---

### 2.1.10 大批量训练的技巧？

**参考答案：**

| 技巧 | 原理 | 效果 |
|------|------|------|
| **梯度累积** | 多步累积后更新 | 等效大batch |
| **LARS/LAMB** | 层自适应学习率 | 支持超大batch |
| **学习率线性缩放** | batch×k则lr×k | 保持收敛性 |

**学习率缩放公式：**
```
lr_new = lr_base × (batch_size_new / batch_size_base)
```

### 2.2.1 如何处理长文本？扩展上下文的技术手段？

**参考答案：**

| 技术 | 原理 | 特点 |

---

### 2.2.2 预训练数据Token重复是否影响模型性能？

**参考答案：**

预训练数据中的 Token 重复可以对模型性能产生一定的影响，具体影响取决于重复的程度和上下文。

**Token 重复的影响：**

| 影响类型 | 具体表现 |
|---------|---------|
| **学习重复模式** | 模型可能学习到重复模式，生成结果缺乏多样性和创造力 |
| **上下文信息不足** | 重复的 Token 可能导致上下文信息缺失，影响复杂语义理解 |
| **训练速度和效率** | 增加计算量和参数数量，增加训练时间和资源消耗 |
| **知识固化** | 过度重复的内容可能被模型视为更重要的知识 |

**处理建议：**
1. **数据去重**：使用 MinHash/LSH 等方法进行去重
2. **控制重复比例**：确保高质量内容的多样性
3. **质量评分**：对重复内容进行过滤或降权

---

### 2.2.3 多语言Tokenizer的构建要点？

**参考答案：**

**关键考虑：**
1. **词汇表大小**：多语言需要更大（100k-250k）
2. **字符覆盖**：确保覆盖所有目标语言字符
3. **语言平衡**：训练时各语言样本均衡
4. **SentencePiece**：首选方案，语言无关

**采样策略（alpha平滑）：**
```
alpha=0: 按数据量采样
alpha=1: 完全平衡
0<alpha<1: 平滑过渡（推荐0.3）
```

## 2.3 训练优化重点

### 2.3.1 AdamW相比Adam的优势？

**参考答案：**

**核心区别：** 权重衰减的实现方式

- **Adam**：L2正则化在梯度上添加 → 影响自适应学习率
- **AdamW**：权重衰减直接作用于参数 → 与自适应学习率解耦

**结果：** AdamW效果更好，是大模型训练的标准选择。

---

### 2.3.2 预训练监控的关键指标？

**参考答案：**

| 指标 | 正常表现 | 异常表现 |
|------|---------|---------|
| **Loss** | 持续下降 | NaN/Inf、震荡大 |
| **PPL** | 10-20 | 过高或不稳定 |
| **梯度范数** | 稳定 | 持续很大或很小 |
| **吞吐量** | 稳定 | 突然下降 |

**异常处理：**
- Loss NaN → 降低学习率、检查数据
- Loss不下降 → 增大学习率、检查数据质量
- 梯度范数大 → 增大裁剪阈值、降低学习率

---

### 2.3.3 Checkpoint策略？

**参考答案：**

**保存内容：**
- 模型参数
- 优化器状态
- 学习率调度器状态
- 训练步数/随机种子

**策略：**
1. **定期保存**：每隔一定步数/token数
2. **保存最佳**：验证集loss最低时
3. **保留最近N个**：节省磁盘空间
