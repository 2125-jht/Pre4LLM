## 一、Transformer 核心机制

### Q1: 详细推导 Self-Attention 的计算过程，时间/空间复杂度是多少？

**答：**

Self-Attention 的计算分为三步：

```
Q = XW_Q,  K = XW_K,  V = XW_V
Attention(Q,K,V) = softmax(QK^T / √d_k)V
```

**详细过程：**
1. **线性变换**：输入 X ∈ R^(n×d) 分别与三个投影矩阵相乘，得到 Q,K,V ∈ R^(n×d_k)
2. **相似度计算**：QK^T ∈ R^(n×n)，计算每两个 token 之间的注意力分数
3. **缩放**：除以 √d_k，防止 softmax 进入饱和区（梯度消失）
4. **Softmax**：按行归一化，得到注意力权重
5. **加权求和**：权重与 V 相乘，得到输出

**复杂度分析：**
- **时间复杂度**：O(n²·d) —— 主要来自 QK^T 的矩阵乘法
- **空间复杂度**：O(n²) —— 需要存储注意力矩阵
- **序列长度瓶颈**：n² 的复杂度是 Transformer 处理长文本的主要障碍

**【面经：字节跳动 2025 届】**
> 面试官追问："如果序列长度是 100K，怎么优化？" → 引出稀疏注意力、Ring Attention、线性注意力等

### Q2: 为什么要除以 √d_k？如果不除会怎样？

**答：**

**原因：** 防止点积结果过大导致 softmax 饱和。

**数学推导：**
- 假设 q,k 的每个元素是独立随机变量，均值为 0，方差为 1
- 点积 q·k = Σ(q_i × k_i)，共 d_k 项相加
- 每项方差 Var(q_i × k_i) = 1，总和方差为 d_k
- 因此 q·k 的标准差为 √d_k

**如果不除 √d_k：**
- d_k 较大时（如 64、128），点积值会很大（方差为 d_k）
- Softmax 输入进入极端区域，梯度变得极小（饱和）
- 反向传播时梯度消失，难以训练

**【延伸追问】**
- "还有哪些方法可以控制 softmax 的数值稳定性？" → LayerNorm、梯度裁剪

### Q3: Transformer 中为什么要用 LayerNorm 而不是 BatchNorm？

**答：**

**核心原因：** NLP 中序列长度不定，BatchNorm 统计不稳定。

| 对比维度 | LayerNorm | BatchNorm |
|---------|-----------|-----------|
| 归一化维度 | 沿特征维度 (d) | 沿 batch 维度 |
| 序列长度敏感 | 不敏感 | 敏感（短序列统计噪声大） |
| 训练/推理一致性 | 无需保存统计量 | 需要保存 running mean/var |
| 并行性 | 完全独立 | 依赖 batch 内所有样本 |

**具体解释：**
1. **序列长度变化**：NLP 中句子长度差异大（10~1000+），BatchNorm 的统计量不稳定
2. **Batch 大小限制**：大模型训练 batch size 受显存限制，BatchNorm 统计量噪声大
3. **推理一致性**：LayerNorm 训练和推理行为完全一致

**Pre-Norm vs Post-Norm：**
```
Post-Norm (原始):  X + Sublayer(LayerNorm(X))  # 深层训练不稳定
Pre-Norm (常用):   X + Sublayer(X) 后再 LN     # 训练更稳定，但可能表达能力略弱
```

**【面经：阿里巴巴 2025 届】**
> 面试官："LayerNorm 的 eps 参数是干什么的？一般设多少？"
> 答：防止除零，默认 1e-5，大模型中有时用 1e-6

### Q4: 详细解释 Multi-Head Attention 的设计动机和计算过程

**答：**

**设计动机：**
1. **多子空间学习**：不同 head 关注不同的语义关系（语法、指代、语义等）
2. **表达能力增强**：多个投影矩阵提供多个表示子空间
3. **并行计算**：各 head 独立计算，可并行

**计算过程：**
```python
# h 个 head，每个维度 d_k = d_model / h
head_i = Attention(XW_Q^i, XW_K^i, XW_V^i)  # 每个 head 独立投影
MultiHead = Concat(head_1, ..., head_h)W_O  # 拼接后线性变换
```

**参数分析：**
- 每个 head：(d_model × d_k) × 3 (Q,K,V)
- h 个 head 总参数量：3 × d_model × d_k × h = 3 × d_model²
- 输出投影：d_model × d_model
- **总计**：4 × d_model²

**【面经：月之暗面 2025 届】**
> 面试官："多个 head 学到的东西真的不同吗？有没有可视化证据？"
> 答：有，BERT/GPT 的可视化显示：部分 head 关注局部语法，部分关注长距离依赖，部分关注指代消解

### Q5: MHA、GQA、MQA 的区别是什么？各自的优缺点？

**答：**

**演进路线：** MHA → GQA → MQA，核心目的是**减少 KV Cache 显存占用**

| 机制 | Q 头数 | K/V 头数 | 参数量 | KV Cache | 代表模型 |
|------|--------|----------|--------|----------|----------|
| MHA | h | h | 4d² | 2bnd | GPT-3, BERT |
| GQA | h | h/g (g个组) | ~3.5d² | 2bnd/g | LLaMA-2, Qwen |
| MQA | h | 1 | 3d² | 2bnd/h | PaLM, StarCoder |

**GQA (Grouped Query Attention)：**
- 多个 query 头共享相同的 K/V 头
- LLaMA-2 采用：8 个 Q 头共享 1 个 KV 头
- **平衡方案**：性能和效率的折中

**MQA (Multi-Query Attention)：**
- 所有 Q 头共享 1 个 K 头 + 1 个 V 头
- **优势**：KV Cache 减少 h 倍，推理显存大幅降低
- **劣势**：表达能力下降，训练时需要调整学习率等超参

**【面经：智谱AI 2025 届】**
> 面试官："GQA 为什么比 MQA 效果更好？"
> 答：保留了一定的表示能力多样性，不会过度压缩信息，LLaMA-2 论文验证了这一点

### Q6: Decoder 的 Masked Attention 是如何实现的？为什么需要 Mask？

**答：**

**因果 Mask 的必要性：**
- Decoder 是自回归生成，位置 i 只能看到之前的位置（0~i-1）
- 防止未来信息泄露（类似时序预测不能用未来数据）

**实现方式：**
```python
# 上三角矩阵，对角线及以上为 -inf
mask = torch.triu(torch.ones(n, n), diagonal=1).bool()
mask = mask.float().masked_fill(mask, float('-inf'))  # 上三角为 -inf

# 加到 attention scores 上
scores = Q @ K.T / sqrt(d_k) + mask  # -inf 位置 softmax 后为 0
attn = softmax(scores) @ V
```

**可视化：**
```
位置:  0   1   2   3
   0 [ 0  -inf -inf -inf]    0 只能看自己
   1 [ 0   0  -inf -inf]     1 看 0,1
   2 [ 0   0    0  -inf]     2 看 0,1,2
   3 [ 0   0    0    0 ]     3 看 0,1,2,3
```

**【延伸追问】**
- "训练和推理时的 Mask 有区别吗？" → 训练用因果 Mask，推理时 KV Cache 隐式实现因果性

## 二、位置编码

### Q7: 绝对位置编码 vs 相对位置编码的区别？各有什么代表方法？

**答：**

| 特性 | 绝对位置编码 | 相对位置编码 |
|------|-------------|-------------|
| 表示方式 | 直接加到输入上 | 在 attention score 中体现相对距离 |
| 长度外推 | 困难 | 天然支持（理论上） |
| 代表方法 | 正弦余弦、可学习 | RoPE、ALiBi、T5 相对偏置 |
| 计算开销 | 低 | 略高 |

**绝对位置编码（原始 Transformer）：**
```python
PE(pos, 2i) = sin(pos / 10000^(2i/d))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d))
```

**关键问题：** 外推性差，训练时 max_len=512，测试时 1000 性能急剧下降

**【面经：百度 2025 届】**
> 面试官："为什么正弦编码的外推性差？"
> 答：训练时模型只见过 [0, 512] 的位置编码，未见过的位置插值效果不佳

### Q8: 详细推导 RoPE (Rotary Position Embedding) 的原理

**答：**

**核心思想：** 通过旋转矩阵编码相对位置，使得 q_m 和 k_n 的内积只与相对位置 m-n 有关。

**数学推导：**

1. **目标**：找到变换 f，使得 f(q,m) · f(k,n) = g(q,k,m-n)

2. **旋转矩阵：**
   对于二维向量 [x,y]，旋转 θ 角度：
   ```
   R(θ) = [cosθ  -sinθ]
          [sinθ   cosθ]
   ```

3. **应用到 attention：**
   将 q,k 每两个维度作为一个子空间，分别旋转：
   ```
   θ_i = 10000^(-2i/d)
   
   f(q,m) = [q_0·cos(mθ_0) - q_1·sin(mθ_0),
             q_0·sin(mθ_0) + q_1·cos(mθ_0),
             ...]
   ```

4. **高效实现（复数形式）：**
   ```python
   def apply_rope(x, cos, sin):
       # x: [batch, heads, seq, dim]
       x1, x2 = x[..., ::2], x[..., 1::2]
       return torch.stack([x1*cos - x2*sin, x1*sin + x2*cos], dim=-1).flatten(-2)
   ```

**关键性质：**
- **相对位置敏感性**：f(q,m)·f(k,n) 只依赖 m-n
- **长度外推性**：旋转角度连续变化，未见过位置可插值/外推

**【面经：DeepSeek 2025 届】**
> 面试官："RoPE 的 base 10000 改大或改小有什么影响？"
> 答：改大（如 1000000）周期变长，支持更长外推；改小周期短，短距离分辨率高

### Q9: ALiBi (Attention with Linear Biases) 的原理和优势？

**答：**

**原理：** 不在输入加位置编码，而是在 attention score 上加上负的线性偏置：
```
score = QK^T / √d_k + mask
其中 mask = -m × |i - j|  (m 是 head 特定的斜率)
```

**Head 特定斜率：**
- 不同 head 使用不同斜率，通常从 2^(-8) 到 2^(-1) 等比分布
- 小斜率 head 关注长距离，大斜率 head 关注短距离

**优势：**
1. **长度外推性强**：无需调整即可从 2K 扩展到 10K+
2. **训练稳定**：不需要额外的位置编码层
3. **推理简单**：无需计算位置编码

**代表模型：** BLOOM、MPT

**【面经：MiniMax 2025 届】**
> 面试官："ALiBi 和 RoPE 哪个外推性更好？"
> 答：ALiBi 原生外推性更好，RoPE 需要配合 NTK-aware、YaRN 等技巧才能达到类似效果

### Q10: NTK-aware 扩展和 YaRN 的原理是什么？

**答：**

**问题背景：** RoPE 从训练长度 L 扩展到 2L，性能会下降。

**NTK-aware 扩展：**
- **核心**：不改变训练好的权重，只修改位置编码的 base
- **原理**：高频分量（短距离）压缩，低频分量（长距离）外推
- **公式**：base' = base × s^(d/(d-2))，s 是扩展倍数
- **优点**：无需微调，即插即用

**YaRN (Yet another RoPE extensioN)：**
- **温度缩放**：对 attention score 除以温度系数 t
- **频率筛选**：只对高频分量进行插值
- **公式**：
  ```
  如果 λ < 阈值：使用插值 (缩小角度)
  如果 λ ≥ 阈值：保持外推 (角度不变)
  ```

**效果对比：**
| 方法 | LLaMA 2 从 4K→32K | 是否需要微调 |
|------|-------------------|-------------|
| 直接外推 | 崩溃 | 否 |
| 线性插值 | 一般 | 推荐微调 |
| NTK-aware | 较好 | 否 |
| YaRN | 最好 | 轻微微调 |

**【面经：零一万物 2025 届】**
> 面试官："如果让你把 4K 的模型扩展到 128K，你会怎么做？"
> 答：YaRN/NTK-aware 初始化 → 长短文本混合数据微调 → Ring Attention 降低显存

## 三、分词与词表

### Q11: BPE、WordPiece、SentencePiece、Unigram 的区别？

**答：**

| 算法 | 合并策略 | 基础单元 | 代表模型 |
|------|---------|---------|---------|
| BPE | 频率最高的相邻 pair 合并 | 字符 | GPT-2, LLaMA, Qwen |
| WordPiece | 最大化似然的 pair 合并 | 字符 | BERT, DistilBERT |
| SentencePiece | BPE/Unigram + 直接处理 raw text | 字符/子词 | T5, ALBERT, XLNet |
| Unigram | 从大到小剪枝 | 所有可能子串 | XLNet, AlBERT |

**BPE 详细过程：**
1. 从字符级词表开始
2. 统计所有相邻 pair 的频率
3. 合并频率最高的 pair，加入词表
4. 重复直到词表达到目标大小

**SentencePiece 优势：**
- 直接处理原始文本（不需要预分词）
- 空格用特殊符号 ▁ 表示
- 支持 BPE 和 Unigram 两种模式

**【面经：腾讯 2025 届】**
> 面试官："为什么 GPT 用 BPE，BERT 用 WordPiece？"
> 答：GPT 是生成模型，BPE 贪心合并简单高效；BERT 需要细粒度编码，WordPiece 基于概率更精确

### Q12: 词表大小如何确定？对模型有什么影响？

**答：**

**常见词表大小：**
| 模型 | 词表大小 | 备注 |
|------|---------|------|
| GPT-2 | 50,257 | BPE |
| LLaMA-2 | 32,000 | 多语言混合 |
| Qwen | 152,064 | 中文优化 |
| Baichuan-2 | 125,696 | 中文 + 多语言 |

**影响分析：**

1. **参数量**：
   - Embedding 层：V × d
   - Output 层：d × V
   - 总增加：2 × V × d
   - 例：V=100K, d=4096 → 额外 0.8B 参数

2. **计算效率**：
   - 词表小 → 序列长（如中文按字分词）
   - 词表大 → 序列短，但 softmax 计算量大

3. **多语言权衡：**
   - 小词表 + 多语言 → OOV 多，编码效率低
   - 大词表 → 增加参数量，但编码效率高

**【面经：商汤科技 2025 届】**
> 面试官："中文大模型为什么词表通常比英文大？"
> 答：中文字符多（几万个），英文用 BPE 几百个 subword 就能覆盖大部分词

## 四、架构演进

### Q13: LLaMA 相比原始 Transformer 做了哪些改进？

**答：**

**LLaMA 的主要改进：**

| 组件 | 原始 Transformer | LLaMA |
|------|-----------------|-------|
| 归一化 | LayerNorm | RMSNorm |
| 激活函数 | ReLU | SwiGLU |
| 位置编码 | 绝对正弦 | RoPE |
| 注意力 | MHA | 改进版 MHA |

**RMSNorm (Root Mean Square Layer Norm)：**
```python
# LayerNorm
y = (x - mean(x)) / sqrt(var(x) + eps) * gamma

# RMSNorm (去掉 mean centering)
y = x / sqrt(mean(x^2) + eps) * gamma
```
- **优势**：计算更快，效果相当

**SwiGLU：**
```python
SwiGLU(x) = (xW + b) ⊗ Swish(xV + c)
# 其中 Swish(x) = x · sigmoid(βx)
```
- 两个线性投影 + 逐元素乘
- 表达能力更强

**【面经：字节跳动 2025 届】**
> 面试官："为什么用 RMSNorm 比 LayerNorm 快？"
> 答：省去了 mean 的计算，只需要一个 reduce 操作

---

### Q14: SwiGLU 激活函数的原理是什么？为什么比 ReLU/GELU 好？

**答：**

**SwiGLU 公式：**
```
SwiGLU(x) = Swish_1(xW) ⊗ (xV)
# Swish_β(x) = x · sigmoid(βx)
```

**为什么效果好：**

1. **门控机制**：类似 LSTM 的门控，选择性传递信息
2. **平滑非单调**：Swish 非单调，ReLU 单调
3. **无界性**：正值无界（ReLU 也是），负值平滑

**与 GELU 对比：**
- GELU: x · Φ(x)，Φ 是标准正态 CDF
- Swish: x · sigmoid(βx)，β=1 时类似 GELU
- SwiGLU 多了一个线性分支，表达能力更强

**【延伸追问】**
- "SwiGLU 有几个权重矩阵？" → 3 个（两个输入投影 + 输出投影）

### Q15: 上下文长度从 4K 扩展到 128K 有哪些关键技术？

**答：**

**技术栈：**

| 层级 | 技术 | 作用 |
|------|------|------|
| 位置编码 | NTK-aware, YaRN, LongRoPE | 支持更长位置 |
| 注意力优化 | FlashAttention-2, Ring Attention | 降低显存到线性 |
| 训练策略 | 长短文本混合、渐进式扩展 | 稳定训练 |
| 推理优化 | KV Cache 压缩、稀疏注意力 | 降低推理成本 |

**渐进式扩展流程：**
```
4K (预训练) 
  → YaRN 扩展到 32K (少量微调)
  → Ring Attention + 长文本数据扩展到 128K
  → 指令微调对齐
```

**Ring Attention：**
- 将序列分块，沿序列维度循环计算 attention
- 显存复杂度从 O(n²) → O(n)（按块）
- 支持理论上无限长序列（受限于计算时间）

**【面经：月之暗面 2025 届 - Kimi 长文本岗】**
> 面试官："100万 token 的上下文，KV Cache 怎么存？"
> 答：KV Cache 压缩（H2O、SnapKV）+ 稀疏 attention（StreamingLLM）+ 分块卸载到 CPU/Disk

### Q16: MoE (Mixture of Experts) 架构的原理和优势？

**答：**

**核心思想：** 用稀疏激活的多个专家网络替代单个密集网络。

**架构：**
```
输入 x → Router → 选择 Top-k 个 Expert → 加权输出

输出 = Σ(g_i(x) · E_i(x))
其中 g_i 是门控分数，E_i 是第 i 个专家
```

**关键设计：**

1. **Top-k 路由**：只激活 k 个专家（通常 k=1 或 2），保持稀疏
2. **负载均衡损失**：防止所有 token 都路由到少数热门专家
   ```
   L_balance = α · Σ(f_i · P_i)
   # f_i: 分配给专家 i 的 token 比例
   # P_i: 门控概率中专家 i 的平均概率
   ```

3. **专家容量**：限制每个专家处理的 token 数，超出溢出

**优势：**
- **容量扩展**：总参数量大增，但激活参数量不变
- **推理成本**：与同等激活参数的密集模型相当
- **训练加速**：专家可并行训练

**代表模型：**
- Switch Transformer (Google)
- GLaM (Google, 1.2T 参数)
- Mixtral 8×7B (Mistral)
- Qwen-MoE

**【面经：DeepSeek 2025 届】**
> 面试官："MoE 的 All-to-All 通信瓶颈怎么解决？"
> 答：专家并行（Expert Parallelism）+ 精细的通信调度 + EP 组划分

## 五、核心概念

### Q17: 什么是 Scaling Laws？它对 LLM 研发有什么指导意义？

**答：**

**Scaling Laws（尺度定律）**：模型性能与计算量、数据量、参数规模之间的幂律关系。

**核心发现（Chinchilla 论文）：**
- **计算最优**：模型参数量 N 与训练 token 数 D 应该 1:1 扩展
- **数据重要性**：过去过度强调参数规模，实际上数据量同等重要
- **例**：70B 模型应该配 1.4T tokens，而非之前常用的 300B

**指导意义：**
1. **资源分配**：给定预算，科学分配参数量和训练数据
2. **性能预测**：小模型实验可外推大模型性能
3. **避免浪费**：防止模型过大而数据不足的"过参数化"

**【面经：智谱AI 2025 届】**
> 面试官："如果只有 1000 张 A100 训练一个月，你会训多大的模型？"
> 答：先算总 FLOPs ≈ 1000 × 30 × 24 × 312 TFLOPS ≈ 2.2e21，根据 Chinchilla 最优，约 30B 参数配 600B tokens

### Q18: 什么是"涌现能力"（Emergent Abilities）？

**答：**

**定义**：小模型不存在，当模型规模超过某个临界点后突然出现的能力。

**典型涌现能力：**
| 能力 | 描述 | 临界规模 |
|------|------|---------|
| In-context Learning | 从 prompt 中的示例学习 | ~10B |
| Chain-of-Thought | 多步推理能力 | ~100B |
| 指令遵循 | 理解复杂指令 | ~100B |

**为什么涌现：**
- **量变引起质变**：足够大的模型容量 + 足够多样的训练数据
- **隐式学习**：通过预测下一个 token 间接学习推理、规划等能力

**争议：**
- 有人认为涌现是评估指标选择的产物（非线性指标）
- 线性指标下能力增长是平滑的

**【面经：OpenAI 2025 届】**
> 面试官："涌现能力对对齐（Alignment）有什么挑战？"
> 答：涌现能力难以预测，可能导致模型突然展现出开发者未预期的行为，增加安全风险

### Q19: LLM 推理有哪些解码策略？各自的优缺点？

**答：**

| 策略 | 原理 | 优点 | 缺点 | 适用场景 |
|------|------|------|------|---------|
| Greedy | 每步选概率最高的词 | 简单、确定 | 容易陷入局部最优、重复 | 确定性任务 |
| Beam Search | 保留 top-k 候选序列 | 全局最优 | 缺乏多样性、计算量大 | 机器翻译、摘要 |
| Temperature | 调整 softmax 温度 | 控制随机性 | 需要调参 | 创意生成 |
| Top-k | 仅从 top-k 中采样 | 避免极低概率词 | k 值难定 | 通用场景 |
| Top-p (Nucleus) | 从累计概率>p的最小集合采样 | 自适应 | 计算略复杂 | **最常用** |

**Temperature 作用：**
- T→0：greedy，确定性
- T=1：原始分布
- T>1：更平坦，更多样

**【面经：阿里通义 2025 届】**
> 面试官："为什么对话模型常用 top_p=0.9，代码生成用 top_p=0.95？"
> 答：代码生成需要更高的确定性，稍大的 p 值保留更多可能选项；对话需要平衡多样性和连贯性

### Q20: L1 和 L2 正则化的区别？什么场景适合使用？

**答：**

| 特性 | L1 正则化 | L2 正则化 |
|------|-----------|-----------|
| 公式 | λΣ\|w\| | λΣw² |
| 效果 | 产生稀疏解（部分权重为0） | 权重平滑，趋近于0但不为0 |
| 用途 | 特征选择 | 防止过拟合 |
| 优化 | 可能多解 | 唯一解 |

**使用场景：**
- **L1**：特征维度高，需要自动特征选择（如稀疏 attention）
- **L2**：深度学习默认选择，Weight Decay 通常指 L2

**在大模型中的应用：**
- LoRA 中的 dropout 可视为隐式正则化
- 大模型本身参数量大，通常 L2 已经足够

### Q21: 训练百亿/千亿参数模型有哪些工程挑战？

**答：**

**三大挑战：**

**1. 显存挑战**
- 问题：千亿模型 + 优化器状态 + 梯度 ≈ 数 TB
- 解决：
  - **3D 并行**：DP + PP + TP
  - **ZeRO**：DeepSpeed 的显存优化，将优化器状态分片
  - **激活重计算**：以时间换空间

**2. 通信挑战**
- 问题：多卡之间梯度同步、激活值传输
- 解决：
  - **NVLink/InfiniBand**：高速互联
  - **通信与计算重叠**：调度优化
  - **梯度压缩**：减少传输量

**3. 训练稳定性**
- 问题：Loss spike、NaN、梯度爆炸
- 解决：
  - **BF16**：比 FP16 更稳定
  - **Pre-LayerNorm**：训练更稳定
  - **梯度裁剪**：防止爆炸
  - **学习率预热**：warmup 阶段

**【面经：华为诺亚 2025 届】**
> 面试官："BF16 和 FP16 的区别？为什么大模型用 BF16？"
> 答：BF16 用 8 位指数（vs FP16 5 位），动态范围更大，不易下溢； mantissa 位少（7 vs 10），精度略低但深度学习通常够用

## 六、关键对比总结

### 位置编码对比

| 方法 | 外推性 | 实现复杂度 | 代表模型 |
|------|--------|-----------|---------|
| 可学习 | 差 | 低 | BERT (早期) |
| 正弦 | 差 | 低 | 原始 Transformer |
| RoPE | 中（需技巧） | 中 | LLaMA, Qwen, Baichuan |
| ALiBi | 好 | 低 | BLOOM, MPT |
| NTK-aware | 好 | 低 | 各类扩展 |
| YaRN | 最好 | 中 | LLaMA 长文本版 |

### 注意力机制对比

| 机制 | KV Cache 大小 | 表达能力 | 适用场景 |
|------|---------------|---------|---------|
| MHA | 2bnd | 最强 | 训练优先 |
| GQA | 2bnd/g | 强 | 平衡选择 |
| MQA | 2bnd/h | 中等 | 极致推理速度 |

### 架构范式对比

| 架构 | 注意力类型 | 代表模型 | 擅长任务 |
|------|-----------|---------|---------|
| Encoder-Only | 双向 | BERT | NLU（分类、NER） |
| Decoder-Only | 单向因果 | GPT, LLaMA | 生成、对话 |
| Encoder-Decoder | 双向+因果+Cross | T5, BART | 翻译、摘要 |

**当前主流**：Decoder-Only 一统江湖（GPT-4, LLaMA, Qwen, Claude 等）
- 原因：结构简单、扩展性好、Causal LM 统一了预训练和指令微调范式
