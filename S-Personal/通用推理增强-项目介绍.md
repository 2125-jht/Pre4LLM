# 大模型通用推理能力增强项目方案

> **项目概述**：本项目基于 **DeepSeek-R1 冷启动策略** 与 **RLPR（Reinforcement Learning with Probability Reward）** 强化学习方法，构建了一个完整的模型通用推理能力增强系统。项目涵盖数据构建、模型训练、评估验证到推理部署的全链路方案，实现了无需外部验证器的通用领域推理能力扩展。

---

## 一、项目背景与技术选型

### 1.1 项目动机

现有的大模型强化学习方法（如 RLVR）主要局限于数学和代码领域，其核心瓶颈在于：
- **验证器依赖**：需要领域特定的规则验证器或训练专门的验证模型
- **扩展性差**：难以推广到开放域的自然语言推理任务
- **工程复杂**：验证器的开发和维护成本极高

本项目旨在解决上述问题，通过结合 DeepSeek-R1 的冷启动数据构建方法和 RLPR 的无验证器强化学习框架，实现通用推理能力的规模化扩展。

class ColdStartDataFilter:
    def __init__(self):
        self.min_think_length = 100    # 推理过程最少字符数
        self.max_think_length = 2000   # 推理过程最多字符数
        self.format_check = True       # 格式检查

### 3.3 RLPR 强化学习训练

RLPR 的核心创新是**使用模型内在的概率作为奖励信号**，无需外部验证器。

#### 3.3.1 RLPR 核心原理

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         RLPR 概率奖励计算流程                                 │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  Input: Question Q + Reference Answer y*                                    │
│                                                                             │
│  Step 1: 生成推理回答                                                        │
│  ┌─────────────────────────────────────────┐                               │
│  │  Policy Model πθ generates:            │                               │
│  │  o = <think>reasoning</think>          │                               │
│  │     <answer>generated_answer</answer>  │                               │
│  └─────────────────────────────────────────┘                               │
│                                                                             │
│  Step 2: 构造概率计算序列                                                    │
│  ┌─────────────────────────────────────────┐                               │
│  │  Replace generated_answer with y*:     │                               │
│  │  o' = <think>reasoning</think>         │                               │
│  │       <answer>reference_answer</answer>│                               │
│  └─────────────────────────────────────────┘                               │
│                                                                             │
│  Step 3: 计算概率奖励                                                        │
│  ┌─────────────────────────────────────────┐                               │
│  │  r = mean(p_i for token in y*)         │                               │
│  │                                        │                               │
│  │  即：参考答案每个 token 的平均解码概率    │                               │
│  └─────────────────────────────────────────┘                               │
│                                                                             │
│  Step 4: 奖励去偏 (Debiasing)                                               │
│  ┌─────────────────────────────────────────┐                               │
│  │  r' = probability(Q + y* without CoT)  │                               │
│  │  r̂ = clip(0, 1, r - r')                │                               │
│  │                                        │                               │
│  │  消除问题和参考答案本身带来的偏差        │                               │
│  └─────────────────────────────────────────┘                               │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

#### 3.3.2 概率奖励实现

```python
# rlpr_reward.py
import torch
import torch.nn.functional as F

class ProbabilityReward:
    """
    RLPR 概率奖励计算器
    
    核心思想：使用模型对参考答案的解码概率作为奖励信号
    """
    
    def __init__(self, policy_model, tokenizer):
        self.policy_model = policy_model
        self.tokenizer = tokenizer
        
    def compute_reward(self, question, generated_response, reference_answer):
        """
        计算概率奖励
        
        Args:
            question: 问题文本
            generated_response: 模型生成的完整回复（包含推理过程）
            reference_answer: 参考答案
        
        Returns:
            debiased_reward: 去偏后的概率奖励 [0, 1]
        """
        # 1. 提取生成的推理过程
        reasoning = self._extract_reasoning(generated_response)
        
        # 2. 构造完整序列：question + reasoning + reference_answer
        full_prompt = self._build_prompt(question, reasoning, reference_answer)
        
        # 3. 计算每个 token 的解码概率
        token_probs = self._get_token_probabilities(full_prompt, reference_answer)
        
        # 4. 计算平均概率（使用 mean 而非 product，更稳定）
        reward = torch.mean(token_probs).item()
        
        # 5. 计算基线概率（无推理过程时的概率）
        baseline_prompt = self._build_prompt(question, "", reference_answer)
        baseline_probs = self._get_token_probabilities(baseline_prompt, reference_answer)
        baseline_reward = torch.mean(baseline_probs).item()
        
        # 6. 去偏并裁剪
        debiased_reward = max(0.0, min(1.0, reward - baseline_reward))
        
        return debiased_reward
    
    def _get_token_probabilities(self, prompt, target_text):
        """获取目标文本每个 token 的解码概率"""
        # 编码输入
        prompt_ids = self.tokenizer.encode(prompt, return_tensors="pt")
        target_ids = self.tokenizer.encode(target_text, return_tensors="pt")
        
        # 拼接
        input_ids = torch.cat([prompt_ids, target_ids], dim=1).to(self.policy_model.device)
        
        with torch.no_grad():
            outputs = self.policy_model(input_ids)
            logits = outputs.logits
            
            # 获取目标位置的概率分布
            target_start = prompt_ids.shape[1]
            target_logits = logits[:, target_start-1:-1, :]  # [batch, seq_len, vocab]
            
            # 计算每个目标 token 的概率
            probs = F.softmax(target_logits, dim=-1)
            token_probs = torch.gather(
                probs, 
                dim=-1, 
                index=target_ids.unsqueeze(-1)
            ).squeeze(-1)
            
        return token_probs[0]  # 返回第一个 batch 的概率序列
```

#### 3.3.3 标准差过滤（训练稳定性）

```python
class StdDevFilter:
    """
    自适应标准差过滤
    
    过滤奖励标准差过低的样本（太简单或太难的题目）
    使用指数移动平均动态调整阈值
    """
    
    def __init__(self, initial_beta=0.5, ema_decay=0.9):
        self.beta = initial_beta  # 过滤阈值
        self.ema_decay = ema_decay
        self.ema_std = None
        
    def should_filter(self, rewards):
        """
        判断是否过滤该 prompt
        
        Args:
            rewards: 该 prompt 的所有采样回复的奖励列表 [r1, r2, ..., rG]
        
        Returns:
            bool: True 表示过滤该 prompt
        """
        std = torch.std(torch.tensor(rewards)).item()
        
        # 更新 EMA
        if self.ema_std is None:
            self.ema_std = std
        else:
            self.ema_std = self.ema_decay * self.ema_std + (1 - self.ema_decay) * std
        
        # 动态调整阈值
        self.beta = 0.5 * self.ema_std
        
        # 标准差过低则过滤
        return std < self.beta
```

```python
# evaluation_metrics.py
class ReasoningEvaluator:
    """推理能力评估器"""
    
    def __init__(self, model, tokenizer):
        self.model = model
        self.tokenizer = tokenizer
        
    def evaluate_benchmark(self, benchmark_data, num_attempts=2):
        """
        评估单个 benchmark
        
        Args:
            benchmark_data: 评估数据集
            num_attempts: 每个问题尝试次数（用于计算 Avg@k）
        
        Returns:
            accuracy: 平均准确率
        """
        correct = 0
        total = 0
        
        for item in benchmark_data:
            question = item["question"]
            reference = item["answer"]
            
            attempts_correct = 0
            for _ in range(num_attempts):
                response = self.generate_response(question)
                predicted = self.extract_answer(response)
                
                if self.check_correctness(predicted, reference):
                    attempts_correct += 1
            
            # Avg@k: 平均尝试成功率
            correct += attempts_correct / num_attempts
            total += 1
        
        return correct / total
    
    def generate_response(self, question, temperature=1.0, max_tokens=3072):
        """生成回复"""
        prompt = self.build_eval_prompt(question)
        
        inputs = self.tokenizer(prompt, return_tensors="pt").to(self.model.device)
        
        with torch.no_grad():
            outputs = self.model.generate(
                **inputs,
                max_new_tokens=max_tokens,
                temperature=temperature,
                do_sample=True,
                pad_token_id=self.tokenizer.pad_token_id,
                eos_token_id=self.tokenizer.encode("</answer>")[0]
            )
        
        response = self.tokenizer.decode(outputs[0][inputs.input_ids.shape[1]:], skip_special_tokens=True)
        return response
    
    def extract_answer(self, response):
        """从回复中提取答案"""
        # 使用正则表达式提取 <answer> 标签内容
        match = re.search(r'<answer>(.*?)</answer>', response, re.DOTALL)
        if match:
            return match.group(1).strip()
        return response.strip()
    
    def check_correctness(self, predicted, reference):
        """
        检查答案正确性
        
        对于不同 benchmark 使用不同的检查策略：
        - 数学/代码：使用规则验证器
        - 通用领域：使用 LLM 作为评判
        """
        # 数学题目使用规则验证
        if self.is_math_problem(reference):
            return self.math_verify(predicted, reference)
        
        # 通用领域使用 LLM 评判
        return self.llm_judge(predicted, reference)
```

# benchmark.py
import time
import asyncio
import aiohttp
import statistics
from concurrent.futures import ThreadPoolExecutor

class PerformanceBenchmark:
    """推理性能基准测试"""
    
    def __init__(self, api_url="http://localhost:8000"):
        self.api_url = api_url
        self.test_prompts = self._load_test_prompts()
    
    def _load_test_prompts(self):

            "Explain the theory of relativity in simple terms",
            "What is the capital of Australia?",
            "Write a Python function to calculate factorial",
            "Compare and contrast Python and Java"
        ]
    
    async def benchmark_latency(self, num_requests=100):
        """测试延迟"""
        latencies = []
        
        async with aiohttp.ClientSession() as session:
            for _ in range(num_requests):
                prompt = random.choice(self.test_prompts)
                start = time.time()
                
                async with session.post(
                    f"{self.api_url}/generate",
                    json={"prompt": prompt, "max_tokens": 512}
                ) as resp:
                    await resp.json()
                
                latency = time.time() - start
                latencies.append(latency)
        
        return {
            "mean_latency": statistics.mean(latencies),
            "p50_latency": statistics.median(latencies),
            "p95_latency": sorted(latencies)[int(len(latencies) * 0.95)],
            "p99_latency": sorted(latencies)[int(len(latencies) * 0.99)],
            "throughput": num_requests / sum(latencies)
        }
    
    async def benchmark_throughput(self, concurrency=10, duration=60):
        """测试吞吐量"""
        completed = 0
        start_time = time.time()
        
        async def worker():
            nonlocal completed
            async with aiohttp.ClientSession() as session:
                while time.time() - start_time < duration:
                    prompt = random.choice(self.test_prompts)
                    async with session.post(
                        f"{self.api_url}/generate",
                        json={"prompt": prompt, "max_tokens": 512}
                    ) as resp:
                        await resp.json()
                    completed += 1
        
        await asyncio.gather(*[worker() for _ in range(concurrency)])
        
        return {
            "total_requests": completed,
            "duration": duration,
            "throughput_rps": completed / duration
        }

### 6.2 面试常见问题

#### Q1: 为什么选择 RLPR 而不是传统的 RLVR？

**回答要点：**
- RLVR 依赖领域特定的验证器，难以扩展到通用领域
- RLPR 使用模型内在概率作为奖励，天然支持开放域任务
- 无需训练额外的验证模型，简化工程实现

#### Q2: 概率奖励为什么比序列似然更稳定？

**回答要点：**
- 序列似然使用概率乘积，单个低概率 token 会导致奖励骤降
- 概率奖励使用平均概率，对异常 token 更鲁棒
- 实验表明平均概率与答案质量的相关性更高（AUC 提升 2-20%）

#### Q3: 标准差过滤的作用是什么？

**回答要点：**
- 过滤太简单（所有回复奖励都很高）或太难（所有回复奖励都很低）的样本
- 使用指数移动平均动态调整阈值，适应训练过程中的分布变化
- 提升训练稳定性，避免模型在无效样本上浪费计算

#### Q4: 冷启动阶段为什么重要？

**回答要点：**
- 纯 RL 训练初期不稳定，模型可能学会错误的推理模式
- 冷启动数据提供高质量的推理示范，引导模型学习正确格式
- 为后续 RL 训练提供更好的初始化点，加速收敛

#### Q5: 如何处理训练过程中的熵坍缩问题？

**回答要点：**
- 使用 clip-high 技巧：裁剪上限设为 1.27 而非 1.2
- 设置熵奖励系数（entropy_coef=1e-3）
- 监控 token 熵，确保其保持在合理范围内（0.4-0.7）
