# 策略梯度与强化学习基础

## 一、策略梯度核心直觉

### 一句话理解策略梯度
**好的行为多做，坏的行为少做。**

想象你是一个学做菜的新手：
- 你随机尝试各种做法（采样）
- 朋友试吃后给打分（奖励）
- 打分高的做法，下次多用（增大概率）
- 打分低的做法，下次少用（减小概率）

这就是策略梯度的全部思想！

| 概念 | 类比 | 数学表达 |
|------|------|----------|
| 策略 | 你做菜的方法 | 概率分布 π(a\|s) |
| 梯度 | 怎么调整这个分布 | ∇θ |
| 奖励 | 朋友的打分 | R |

**核心公式预告：**
```
梯度 = 奖励 × log概率的梯度
∇J(θ) = E[ R · ∇log π(a|s; θ) ]
```

## 二、SFT vs 策略梯度

### SFT的梯度（你已经很熟了）
```
∇L_SFT = -∇log π(a*|s; θ)
```
对每个标准答案 a*，增大其概率

### 策略梯度只多了一个「权重」
```
∇J_PG = -R · ∇log π(a|s; θ)
```
- a 不是标准答案，而是模型自己采样的！
- R 是这个采样结果的奖励分数

### 记忆口诀
| 方法 | 特点 |
|------|------|
| SFT | 老师说什么就学什么（有标准答案） |
| PG | 自己试，好的多学坏的少学（试错学习） |

**区别：** SFT的权重恒为1（每条数据同等重要），策略梯度的权重是R（好数据权重大、坏数据权重小甚至为负）。

## 三、REINFORCE：最简策略梯度算法

只需4步，就能实现策略梯度：

| 步骤 | 操作 |
|------|------|
| Step 1 | 用当前策略 π(θ) 采样一条轨迹 |
| Step 2 | 计算这条轨迹的总奖励 R |
| Step 3 | 计算梯度 g = R · ∇log π(a\|s;θ) |
| Step 4 | 更新参数 θ ← θ + α · g |

### 直觉解读每一步
- **∇log π(a\|s;θ)** 指明了「增大动作a概率」的方向
- **乘以 R > 0**：沿这个方向走 → 增大好动作概率
- **乘以 R < 0**：反方向走 → 减小坏动作概率
- **R = 0**：不更新（中性结果不提供信息）

### 问题
**方差极大！** 一条轨迹的R波动很大，梯度估计不稳定。→ 需要baseline和更好的advantage估计。

---

## 四、从R到Advantage：降方差的关键

原始REINFORCE用总奖励R，方差巨大。三步优化：

### 第一板斧：引入Baseline b
```
∇J = E[ (R - b) · ∇log π ]
```
b 通常取奖励均值。直觉：**不是「好不好」而是「比平均好不好」**

### 第二板斧：Advantage函数 A(s,a)
```
A(s,a) = Q(s,a) - V(s)
       = 这个动作的价值 - 这个状态的平均价值
```
直觉：这个动作「比平均水平好多少」

### 第三板斧：GAE(λ) 广义优势估计
```
A_t^GAE = Σ(γλ)^l · δ_{t+l}
δ_t = r_t + γV(s_{t+1}) - V(s_t)
```
λ控制偏差-方差权衡（0→低方差, 1→低偏差）

### 最终公式
```
∇J = E[ A(s,a) · ∇log π(a|s;θ) ]
```
这就是PPO/GRPO等所有方法的共同基础！

## 五、策略梯度家族进化图

所有方法都基于同一个核心：**A · ∇log π**

| 算法 | 梯度公式 | 特点 |
|------|----------|------|
| **REINFORCE (1992)** | 梯度 = R · ∇log π | 最原始，方差大，没有critic |
| **Actor-Critic (A2C)** | 梯度 = A(s,a) · ∇log π | 引入Critic网络估计V(s)，降低方差 |
| **TRPO → PPO (2017)** | 梯度 = min(rA, clip(r,1±ε)A)<br>r=π_new/π_old | 限制更新步长防崩溃<br>PPO就是TRPO的简化实用版 |
| **GRPO (DeepSeek, 2024)** | 去掉Critic，用组内相对排序估计A<br>A_i = (r_i - μ) / σ | 更适合LLM场景 |
| **DAPO (ByteDance, 2025)** | 在GRPO基础上加4个trick<br>非对称clip + 动态采样 + token-level loss | |

## 六、面试高频问答速查

### Q: 策略梯度的核心思想？
**A:** 增大高奖励动作的概率，减小低奖励动作的概率。通过采样估计梯度方向，用奖励信号加权。

### Q: 为什么要用log概率？
**A:** 因为 ∇log π = ∇π/π，这个形式允许我们用采样均值来近似期望（重要性采样的基础），且数值稳定。

### Q: Advantage比R好在哪？
**A:** 减去baseline后，梯度方差大幅降低，不改变梯度期望（无偏估计）。A>0增大概率，A<0减小概率，比「绝对好坏」更有效。

### Q: PPO的clip有什么用？
**A:** 限制新旧策略的偏离程度。r(θ)超出[1-ε,1+ε]时梯度被截断，防止一次更新太猛导致策略崩溃。ε通常0.1~0.2。

### Q: GRPO和PPO最大的区别？
**A:** GRPO去掉了Critic网络，用同prompt的多个采样的组内归一化奖励作为Advantage。省50%显存，适合LLM场景。代价是Advantage估计不如GAE精确。
